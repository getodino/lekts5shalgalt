# Лекцийн сэдэв

## Асуулт
Hash + Salt технологийг ашигласнаар халдагчид нууц үгийг тайлахад яагаад хүндрэлтэй болдог вэ?

## Хариулт
Hash + Salt технологийн тайлбар
Hash гэдэг нь нууц үгийг математик аргаар өөр хэлбэрт хөрвүүлэх процесс юм. Жишээ нь:

password123 → ef92b778bafe771207

Salt гэдэг нь нууц үгэнд нэмэгддэг санамсаргүй өгөгдөл юм. Жишээ нь:

password123 + x7Kp9 → hash хийгдэнэ


Яагаад хүндрэлтэй болдог вэ?
1. Rainbow Table халдлага ажиллахгүй

Salt байхгүй үед халдагчид урьдчилан тооцоолсон hash хүснэгт ашиглан нууц үгийг олдог
Salt нэмэгдсэнээр тэр хүснэгт ашиггүй болдог

2. Ижил нууц үг өөр hash үүсгэдэг

Хэрэглэгч А: password123 + saltA → hash1
Хэрэглэгч Б: password123 + saltB → hash2
Тиймээс нэгийг тайлсан ч нөгөөг тайлж чадахгүй

3. Brute Force удаашруулдаг

Халдагч тухайн хэрэглэгчийн salt-ийг мэдэж байж hash тооцоолох ёстой
Энэ нь халдлагын хугацааг эрс уртасгадаг
